# Obsidian Quick Memo 插件设计

日期：2026-06-18

## 目标

构建一个 Markdown-native 的 Obsidian 快速记录总览插件，用于记录日常行为、闪念和待办。插件以 Daily Note Markdown 文件作为唯一真源，只在 `## Quick Memo` 区域中读写记录，并维护一个可重建缓存索引用于历史查看、标签/关键词检索和热力图统计。

首版以三栏总览页面作为主入口，提供快速输入、记录流、过滤器和热力图。插件优先复用 Daily Notes / Calendar 的日期文件配置，缺失时使用插件内的兜底配置。

## 产品范围

### 首版包含

- 总览页面作为主入口，可通过 Ribbon 图标、命令面板和热键打开。
- 三栏布局：
  - 左侧：用户头像、名称、slogan、类型过滤、标签过滤、日期范围、关键词搜索。
  - 中间：Markdown 输入区、类型选择、保存按钮、记录流。
  - 右侧：每日记录数量热力图。
- 三类记录：
  - `记录`：普通日常记录。
  - `闪念`：突然出现的灵感或想法。
  - `待办`：Markdown task，支持完成/未完成切换。
- 支持 Markdown 和多行内容记录。
- 写入 Daily Note 的 `## Quick Memo` 区域。
- 支持编辑记录内容、类型和标签。
- 支持删除单条记录，并在删除前确认。
- 支持打开源文件定位到记录附近。
- 支持复制块链接；无 ID 记录提示需要补全 ID。
- 支持标签过滤、类型过滤、日期范围过滤和关键词检索。
- 支持以当天记录流/时间线形式查看每日时序。
- 支持索引重建。
- 支持设置用户资料、兜底日期文件配置、Quick Memo 标题、是否启用块 ID、默认记录类型和排序方式。

### 首版不包含

- 不做完整 Thino 兼容或数据迁移。
- 不做独立数据库真源。
- 不索引整篇 Daily Note 或全库任意 Markdown 内容。
- 不做提醒、打卡、连续天数。
- 不做复杂统计仪表盘。
- 不做多用户云同步。
- 不做移动端专项 UI，但保持基础响应式可用。
- 不做记录类型完全自定义，只预留扩展点。

## 数据格式

每日笔记中的 `## Quick Memo` 区域是唯一数据源。插件只管理该标题下的列表内容，不修改日记其他区域。

默认 Markdown 格式：

```md
## Quick Memo

- 09:12 [闪念] 插件总览页布局可以做成三栏 #obsidian ^oqm-20260618-091200-a1b2
  中间是输入区和记录流。
  右侧是热力图。
- 10:05 [记录] 今天开始整理插件需求 #project ^oqm-20260618-100500-c3d4
- [ ] 10:20 [待办] 写第一版设计文档 #todo ^oqm-20260618-102000-e5f6
```

规则：

- 普通记录和闪念使用普通列表项：`- HH:mm [类型] 内容 ^oqm-id`。
- 待办使用任务列表项：`- [ ] HH:mm [待办] 内容 ^oqm-id`。
- 完成待办使用：`- [x] HH:mm [待办] 内容 ^oqm-id`。
- 多行 Markdown 内容写成该列表项的缩进子内容。
- 标签使用 Obsidian 原生 `#tag`，由解析器从内容中提取。
- 块 ID 默认启用，格式为 `^oqm-YYYYMMDD-HHmmss-xxxx`。

## 块 ID 与纯净 Markdown 模式

插件默认启用稳定编辑模式，为每条新记录写入 Obsidian 原生块 ID。这样可以稳定定位记录，支持可靠编辑、待办勾选、块链接复制和历史追踪。

同时，插件提供纯净 Markdown 模式：

- 关闭块 ID 后，新记录不写入 `^oqm-*`。
- 无 ID 记录仍可被解析、检索和展示。
- 对无 ID 记录进行编辑、勾选或删除时，插件先尝试使用文件路径、日期、时间和内容摘要进行弱定位。
- 如果弱定位不唯一，插件阻止危险修改，并提示用户补全 ID 或手动处理源文件。
- 插件提供命令为当前或全部 `## Quick Memo` 区域补全缺失 ID。

## 写入与解析规则

### 写入新记录

保存新记录时：

1. 根据当前日期解析目标 Daily Note 文件路径。
2. 如果文件不存在，创建文件。
3. 如果文件缺少 `## Quick Memo` 标题，默认追加到文件末尾。
4. 将新记录追加到 `## Quick Memo` 区域末尾。
5. 如启用块 ID，生成唯一块 ID 并写入行尾。
6. 更新内存索引；必要时触发局部重建。

### 解析记录

解析器按文件日期和 `## Quick Memo` 标题定位区域，并解析列表项：

- 识别记录 ID：`^oqm-*`。
- 识别类型：`[记录]`、`[闪念]`、`[待办]`。
- 识别时间：`HH:mm`。
- 识别待办状态：`- [ ]` 或 `- [x]`。
- 收集缩进子内容作为同一条记录的 body。
- 提取标签和纯文本关键词索引用内容。

用户手动编辑 Markdown 时：

- 如果 ID 保留，插件继续稳定追踪该记录。
- 如果 ID 缺失但格式符合，记录仍进入索引，但编辑/勾选前可能要求补全 ID。
- 如果格式损坏，该条记录不进入索引，解析器返回 warning；其他记录继续正常显示。
- 如果同一个 ID 重复出现，索引标记冲突；编辑时提示用户修复或确认处理。

### 编辑、勾选和删除

- 编辑记录内容时，通过块 ID 或弱定位找到对应列表项，替换首行内容和缩进 body，保留原创建时间和 ID。
- 切换待办状态时，只修改 `- [ ]` / `- [x]`。
- 如果待办被手动改成普通记录，插件按当前 Markdown 格式显示，不强行恢复。
- 删除记录前必须确认。删除只移除该列表项及其缩进子内容，不删除其他 Daily Note 内容。

## 总览页面交互

### 左侧栏：个人与过滤区

- 用户头像、名称、slogan 从插件设置读取。
- 类型过滤：全部、记录、闪念、待办、已完成待办、未完成待办。
- 标签列表：从 Quick Memo 索引提取，按使用频次或字母排序。
- 日期范围过滤：今天、最近 7 天、最近 30 天、自定义范围。
- 关键词搜索框：检索记录正文、缩进内容和标签文本。

### 中间主栏：输入与记录流

- 顶部是多行 Markdown 输入框。
- 类型选择：记录、闪念、待办。
- 保存按钮支持点击和 `Cmd/Ctrl + Enter` 快捷提交。
- 保存后写入当天 Daily Note 的 `## Quick Memo` 区域，清空输入框，并刷新记录流。
- 默认展示今天记录。
- 记录流默认按时间降序显示，最新记录在上；排序可在设置中调整。
- 记录卡片显示时间、类型、正文、标签和待办状态。
- 待办可直接勾选完成/未完成。
- 每条记录支持编辑、删除、复制块链接、打开源文件。

### 右侧栏：热力图

- 热力图按每天 Quick Memo 记录总条数统计颜色深浅。
- 鼠标悬停显示日期和记录数量。
- 点击某一天后，中间记录流切换到该日期。
- 首版只统计全部记录，预留后续按类型切换。

### 每日时序图

每日时序图首版作为中间记录流的一种展示方式：

- 当选择某一天时，按记录时间展示当天时间线。
- 记录卡片左侧显示时间。
- 无记录日期显示空状态和快速记录入口。
- 后续可扩展为更明显的竖向时间轴或小时分布图。

## 内部架构

### Plugin bootstrap / main

- 注册 Ribbon 图标、命令、设置页和总览 View。
- 加载和保存插件设置。
- 监听文件变更事件，触发索引更新。
- 只做装配，不承载业务逻辑。

### SettingsService

管理插件设置：

- 用户名称、slogan、头像路径或 URL。
- Quick Memo 标题，默认 `Quick Memo`。
- 兜底日记文件夹和日期格式。
- 是否启用块 ID，默认启用。
- 默认记录类型。
- 记录流排序方式。

### DailyNoteResolver

负责找到目标日期文件：

- 优先读取 Daily Notes / Calendar 配置。
- 缺失时使用插件兜底配置。
- 根据日期生成文件路径。
- 文件不存在时创建。
- 确保文件中存在 `## Quick Memo` 区域。

### MarkdownRecordRepository

所有对 Markdown 真源的读写都通过该模块：

- `appendRecord(date, recordDraft)`
- `updateRecord(recordIdOrLocator, changes)`
- `toggleTodo(recordIdOrLocator)`
- `deleteRecord(recordIdOrLocator)`
- `readRecords(dateOrRange)`
- `backfillMissingIds(scope)`

该模块负责文件级修改，不负责 UI。

### QuickMemoParser

负责 Markdown 解析和序列化：

- 从 `## Quick Memo` 区域解析记录。
- 解析类型、时间、任务状态、ID、标签、多行 body。
- 将记录对象序列化回列表项 Markdown。
- 对格式异常返回 warnings，不直接弹 UI。

### IndexService

维护可重建缓存索引：

- 启动时懒加载或后台构建索引。
- 文件变更时局部更新。
- 支持按日期、标签、类型和关键词查询。
- 为热力图提供每日记录数量聚合。
- 缓存只是性能优化，不作为真源。

索引数据可保存在插件 data 中，包括文件 mtime、记录摘要、标签计数和每日记录条数。缓存损坏时，插件丢弃缓存并从 Daily Notes 文件重建，不影响 Markdown 原始记录。

### QuickMemoView

总览页面 View：

- 渲染左侧个人资料和过滤器。
- 渲染中间输入区和记录流。
- 渲染右侧热力图。
- 调用 Repository 和 IndexService，不直接改文件。
- 管理 UI 状态：当前日期、筛选条件、编辑中的记录等。

### SettingsTab

插件设置界面：

- 配置用户资料。
- 配置 Daily Notes 兜底路径和日期格式。
- 配置 Quick Memo 标题。
- 配置是否启用块 ID。
- 配置排序和默认类型。

## 数据对象

```ts
type QuickMemoType = "record" | "flash" | "todo";

interface QuickMemoRecord {
  id?: string;
  date: string;
  time: string;
  type: QuickMemoType;
  content: string;
  body?: string;
  tags: string[];
  completed?: boolean;
  filePath: string;
  lineStart: number;
  lineEnd: number;
  hasStableId: boolean;
  raw: string;
}
```

`id` 是可选字段，用于支持纯净 Markdown 模式和历史无 ID 内容。

## 错误处理

- 找不到 Daily Notes 配置：使用插件兜底配置。
- 文件不存在：创建文件并插入标题。
- 文件写入失败：显示 Notice，并保留用户输入。
- 解析失败：跳过异常条目，其他记录正常显示。
- 无 ID 且定位不唯一：阻止危险编辑，提示补全 ID。
- 重复 ID：展示警告，编辑时要求用户确认或先修复。
- 删除记录：必须确认，避免误删 Daily Note 内容。

## 测试策略

首版重点测试非 UI 逻辑：

- Parser：单行、多行、任务、标签、无 ID、重复 ID、异常格式。
- Repository：追加、更新、勾选、删除、保留其他日记内容。
- DailyNoteResolver：Daily Notes 配置优先、兜底配置、文件创建。
- IndexService：按日期、类型、标签、关键词查询，热力图聚合。
- View 层：轻量交互测试或手动验收清单。

## 验收标准

1. 首次使用时，用户能设置名称、slogan、头像和兜底日记路径；打开总览页能看到空状态和输入区。
2. 用户选择“闪念”，输入多行 Markdown 并保存后，当天 Daily Note 出现 `## Quick Memo` 和对应列表项，总览页立即显示该记录。
3. 用户选择“待办”保存后，Daily Note 写入 `- [ ] HH:mm [待办] 写第一版设计文档 #todo ^oqm-20260618-102000-e5f6`；在总览页勾选完成后，文件变成 `- [x] HH:mm [待办] 写第一版设计文档 #todo ^oqm-20260618-102000-e5f6`。
4. 用户编辑已有闪念内容并保存后，Daily Note 对应列表项更新，块 ID 和创建时间保留。
5. 多天有记录时，右侧热力图颜色深浅反映记录数量；点击某天，记录流展示该天时间线。
6. 关键词检索只返回 Quick Memo 区域内匹配记录；标签和类型过滤结果正确。
7. 关闭块 ID 后，新记录不写 `^oqm-*`，但仍能显示和检索；对无 ID 记录执行不安全编辑时，插件提示补全 ID或无法唯一定位。
8. 删除插件缓存后，插件能从 Daily Notes 文件重新构建历史、标签和热力图数据。

## 设计决策总结

- 方案：Markdown-native 总览插件。
- 真源：Daily Note Markdown 文件。
- 缓存：可重建索引。
- 写入位置：每日文件 `## Quick Memo` 区域。
- 存储格式：统一列表项 + `[记录]/[闪念]/[待办]` + 可选块 ID。
- 主入口：三栏总览页面。
- 首版重点：记录、编辑、待办、历史、筛选、热力图。
